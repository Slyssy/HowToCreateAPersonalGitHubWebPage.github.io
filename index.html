Create a Personal Page on GitHub

Step1: Create a GitHub Account

  1. Log onto GitHub by going to www.github.com
  2. Create a GitHub account by filling out the Username, Email and Password.
  3. Click "Sign Up".

Step2: Create a New Repository

  1. Click on the green box that says "New" in the repositories section. This should take you to a page that looks like this.
  2. Enter a repository name.
  3. When naming your repository, use the following syntax. [yourusername].github.io

    4. Note: You should always include a ReadMe file, and if you want to add a gitignore file you can select the checkbox. If you create .gitignore file, you will need to select a template. After you select your option click "Create repository". That will take you to a page like this.

Step3: Clone Your New Repository to Your Local Machine

  1. At this point, you simply need to select the green "Code" button. That will pop up a dialogue box like this.
  2. Copy the SSH Link.
  3. Now you will need to open your terminal and navigate to where you want your directory on your local machine.
  4. In your terminal, type git clone [paste the ssh link you copied] and press "Enter". Your terminal should look like this.

Step 4: Create Files and Open VSCode

  1. Back in your terminal create a new index.html file by typing touch index.html. (Note: Make sure you are in the correct file in your terminal)
  2. Now open the folder you just cloned in VS Code. Your VS Code screen should look something like this.
  3. You can create the HTML template by typing ! then hitting [enter] on the first line of you index.html file.
  4. Build an awesome personal web page.

Step 5: Upload Changes to GitHub and Deploy Your Personal Page

Now that your files are created, you need to push the changes to your GitHub Repository. This can be done by first opening the terminal in VSCode.

VSCode makes it really easy to access the terminal. While in VSCode simply hold down [control] + `
  1. Now that your terminal is open, type git status in your terminal. Your terminal should look something like this.

    2. Note: This will tell you what files have been changed or added on your local machine that are different from what are in the GitHub repository. The updated files will appear in red.

  2. Now type git add .
  3. This adds the files to the queue to be committed. To confirm the files were added to the queue, type git status. Your terminal should look like this.

    4. Note: the file names are now green.

  4. Next, you need to commit the files. This is done by typing git commit -m"Initial Commit"

    5. Note: All commits must include a message to explain what is being committed. That is what -m" " flag indicates. You can type what ever you want between the quotation marks.

    Your terminal should now look like this...

  5. Finally, we need to push the changes to the GitHub repository. This is done by typing git push.

    6. Your terminal should now look something like this...

  6. Check your GitHub repository to confirm the changes were uploaded. Your repository should look something like this...

    7. Note: Make sure the green "Active" notification is displayed in the Environments section.

  7. You are DONE! Go to your browswer and type in the name you created back in step 2 to access your deployed GitHub personal page.
